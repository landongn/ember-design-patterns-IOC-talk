Design Patterns help turn this:

I started out with some code

Then created an Object

Then a Service that created Objects

Then wrote some initializer and injected that Service

Then everything broke

Now I have an AbstractFactoryInjectionServiceObject

Oh god, what have I done

Into something like this:

I create a Service

I define some computed properties around my data

I create a Component

I inject my service into the component

I define computed aliases for the data I need

Basics

Services should be designed to do one thing.

Services shouldn't know about other services.

Services can depend on other modules...

...So long as those modules are well encapsulated.

Build Models

Ember doesn't have traditional MVC "models".

Services are glue between views and data.

They should also help you model the features your app provides

Properties:


import Ember from 'ember';

export default Ember.Service.extend({

  productsInBundle: Ember.computed('products.[]', function () {
    return this.get('products').mapBy('bundle', true);
  }),

  productsOnSale: Ember.computed('products.[]', function () {
    return this.get('products').mapBy('sale', true);
  })

});

Component code:

Definition:


import Ember from 'ember';

export default Ember.Component.Extend({

  products: Ember.inject.service(),

  willInsertElement() {
    this.setProperties({
      onSale: this.products.get('productsOnSale'),
      inBundle: this.products.get('productsInBundle')
    })
  }
});
