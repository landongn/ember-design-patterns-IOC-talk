White-box vs Black-box

White Boxes make you open 'em up and learn the internals

Black Boxes make you slot in your user-land code into the framework

White box feels like:

I started out with some code

Then created an Object

Then a Service that created Objects

Then wrote some initializer and injected that Service

Then everything broke

Now I have an AbstractFactoryInjectionServiceObject

Black box feels like:

I create a Service

I define some computed properties around my data

I create a Component

I inject my service into the component

I define properties from the service for the feature I need

Services

Services should be designed to do one thing.

Services shouldn't know about other services.

Services can depend on other modules...

...So long as those modules are well encapsulated.

Build Models

No, not those kinds of models.

Services are models for features.

They work best when they fully encapsulate a white-box feature.

Example: Shopping Cart

a Shopping Cart should be a service.

A cart contains Products, fetched from an API

Products have some metadata, such as price and sale discounts.

What would that look like?

Cart Service

Definition:


import Ember from 'ember';

export default Ember.Service.extend({

  products: Ember.A(),
  inCart: Ember.A(),
  subtotal: Ember.computed('inCart.[]', function () {},
  taxRate: 0.03,
  productsInBundle: Ember.computed('products.[]', function () {
    return this.get('products').mapBy('bundle', true);
  }),

  productsOnSale: Ember.computed('products.[]', function () {
    return this.get('products').mapBy('sale', true);
  }),

  init() {
    // http://babeljs.io/docs/usage/experimental/ YAY
    this.get('products').pushObjects(await fetchCartForUser());
    this.get('inCart').pushObjects(await fetchCartForUser());
  }
});

Why is this useful?

Fetching the full products array is heavy for components

Greedy-binding updates make your UI more snappy

Populate a component earlier in the lifecycle

Less coupling between services and UI code

Component code:

Definition:


import Ember from 'ember';

export default Ember.Component.Extend({

  products: Ember.inject.service(),

  willInsertElement() {
    this.setProperties({
      onSale: this.products.get('productsOnSale'),
      inBundle: this.products.get('productsInBundle')
    })
  }
});
